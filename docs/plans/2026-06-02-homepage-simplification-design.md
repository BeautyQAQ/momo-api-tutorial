# 首页精简设计

## 目标

把首页从“说明集合页”收紧为“配置入口页”，减少长段公告、计费解释和重复提示对用户注意力的干扰。

## 推荐方案

采用轻量精简方案：保留首屏、快速入口和 Base URL 复制能力；把首页长表格、长公告列表、阅读顺序和重复提示改成短入口与少量关键提醒。详细说明继续放在公告页、快速开始页和排错页。

## 首页结构

1. 首屏：说明站点用途，主按钮进入快速开始，次按钮进入排错。
2. 四个短卡片：快速开始、Codex CLI、Claude Code、Trae / IDE。
3. 新人入口：注册、充值、兑换、创建 API 令牌、复制 Base URL。
4. 关键提醒：只保留地址规则、计费风险、服务波动三类提醒，每条给出详情页链接。

## 不做

- 不新增复杂组件。
- 不改变公告页、快速开始页、客户端教程页内容。
- 不引入新视觉风格。
